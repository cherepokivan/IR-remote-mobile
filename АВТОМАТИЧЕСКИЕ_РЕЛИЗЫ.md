# 🚀 Автоматические релизы

## Как это работает

Теперь релиз создаётся **автоматически при каждом push в main**!

### При push в main:
```bash
git push origin main
```

GitHub Actions автоматически:
1. ✅ Соберёт release APK
2. ✅ Создаст тег с версией (например: `v1.0.0-dev-20260430-194655`)
3. ✅ Создаст GitHub Release
4. ✅ Прикрепит APK файл: `IR-Remote-v1.0.0-dev-20260430-194655.apk`
5. ✅ Пометит как pre-release (жёлтая метка)

### При создании тега вручную:
```bash
git tag v1.0.0
git push origin v1.0.0
```

GitHub Actions:
1. ✅ Соберёт release APK
2. ✅ Создаст GitHub Release с тегом `v1.0.0`
3. ✅ Прикрепит APK: `IR-Remote-v1.0.0.apk`
4. ✅ Пометит как stable release (зелёная метка)

## Формат версий

### Автоматические (при push в main):
```
v1.0.0-dev-20260430-194655
│  │ │  │   │        └─ Время (ЧЧ:ММ:СС)
│  │ │  │   └─ Дата (ГГГГММДД)
│  │ │  └─ dev (разработка)
│  │ └─ Patch версия
│  └─ Minor версия
└─ Major версия
```

### Ручные теги (стабильные релизы):
```
v1.0.0
│ │ │
│ │ └─ Patch (исправления)
│ └─ Minor (новые функции)
└─ Major (крупные изменения)
```

## Где найти релизы

1. Откройте GitHub репозиторий
2. Перейдите в раздел **Releases** (справа)
3. Скачайте последний APK файл

Или прямая ссылка:
```
https://github.com/ВАШ_USERNAME/IR-Remote/releases
```

## Типы релизов

### 🟡 Pre-release (автоматические)
- Создаются при каждом push в main
- Помечены как "Pre-release"
- Для тестирования новых функций
- Имя: `IR-Remote-v1.0.0-dev-20260430-194655.apk`

### 🟢 Stable (ручные теги)
- Создаются при push тега
- Помечены как "Latest"
- Стабильные версии для пользователей
- Имя: `IR-Remote-v1.0.0.apk`

## Примеры использования

### Разработка (автоматические релизы):
```bash
# Просто пушьте в main
git add .
git commit -m "feat: add new device support"
git push origin main

# Релиз создастся автоматически через 3-4 минуты
```

### Стабильный релиз:
```bash
# Создайте тег
git tag v1.0.0
git push origin v1.0.0

# Стабильный релиз создастся автоматически
```

## Описание релиза

Каждый релиз содержит:
- 📦 Версию
- 🔗 Commit SHA
- 📝 Инструкцию по установке
- 📱 Требования (Android 5.0+)
- 📥 APK файл для скачивания

## Проверка статуса сборки

1. Откройте GitHub репозиторий
2. Перейдите в **Actions**
3. Найдите workflow "Release Build"
4. Посмотрите статус (✅ зелёная галочка = успешно)

## Отключение автоматических релизов

Если хотите создавать релизы только вручную, удалите эти строки из `.github/workflows/release.yml`:

```yaml
on:
  push:
    branches:
      - main  # ← Удалить эти 2 строки
```

Оставьте только:
```yaml
on:
  push:
    tags:
      - 'v*'
```
