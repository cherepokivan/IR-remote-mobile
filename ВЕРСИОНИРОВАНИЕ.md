# 📦 Автоматическое версионирование APK

## Как это работает

Версия APK теперь автоматически определяется из git:

### versionCode (число)
Количество коммитов в репозитории:
```bash
git rev-list --count HEAD
```
Например: 1, 2, 3, 4... (увеличивается с каждым коммитом)

### versionName (строка)
Последний git тег или commit hash:
```bash
git describe --tags --always --dirty
```

## Примеры версий

### Без тегов:
- `versionCode`: 15 (15 коммитов)
- `versionName`: "a3f2b1c" (короткий commit hash)

### С тегами:
```bash
git tag v1.0.0
git push --tags
```
- `versionCode`: 15
- `versionName`: "1.0.0"

```bash
git tag v1.1.0
git push --tags
```
- `versionCode`: 23
- `versionName`: "1.1.0"

### После тега (есть новые коммиты):
```bash
git tag v1.0.0
# делаем 3 коммита
```
- `versionCode`: 18
- `versionName`: "1.0.0-3-ga3f2b1c" (тег + 3 коммита + hash)

## Как создать релиз

### 1. Создайте тег с версией:
```bash
git tag v1.0.0
git push origin v1.0.0
```

### 2. GitHub Actions автоматически:
- Соберёт release APK
- Создаст GitHub Release
- Прикрепит APK к релизу

### 3. Файл будет называться:
```
app-release-v1.0.0.apk
```

## Схема версионирования (рекомендуется)

### Semantic Versioning: `MAJOR.MINOR.PATCH`

- **MAJOR** (1.x.x) - несовместимые изменения API
- **MINOR** (x.1.x) - новые функции (обратно совместимые)
- **PATCH** (x.x.1) - исправления багов

### Примеры:
```bash
v1.0.0  # Первый релиз
v1.0.1  # Исправлен баг с IR передатчиком
v1.1.0  # Добавлена поддержка новых устройств
v2.0.0  # Полностью переработан UI
```

## Текущая версия

Проверить текущую версию:
```bash
git describe --tags --always
```

Если тегов нет, будет показан commit hash: `a3f2b1c`

## Создание первого релиза

```bash
# 1. Закоммитить все изменения
git add .
git commit -m "fix: correct IRDB URLs, add IR diagnostics, fix test reporter"

# 2. Создать тег
git tag v1.0.0

# 3. Запушить с тегами
git push origin main --tags
```

GitHub Actions автоматически соберёт и опубликует релиз! 🚀

## Проверка версии в APK

После установки APK:
1. Настройки → Приложения → IR Remote
2. Версия будет показана внизу

Или в коде:
```kotlin
val versionName = BuildConfig.VERSION_NAME
val versionCode = BuildConfig.VERSION_CODE
```

## Fallback

Если git недоступен (например, скачали ZIP без истории):
- `versionCode`: 1
- `versionName`: "1.0.0"

---

**Готово!** Теперь версия APK будет автоматически увеличиваться с каждым коммитом и тегом. 📦
